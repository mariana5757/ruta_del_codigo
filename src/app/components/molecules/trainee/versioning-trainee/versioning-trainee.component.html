📘 ¿Qué es GIT?

Git es un sistema de control de versiones distribuido que permite gestionar y rastrear cambios en el código de un proyecto. Su propósito es facilitar la colaboración entre desarrolladores, mantener un historial de cambios y revertir errores en el código cuando sea necesario.

⚙️ Configuración inicial

  1. Regístrate en GitHub.
  2. Descarga e instala Git en tu equipo desde git-scm.com.
  3. Configura tu nombre y correo: 
    git config --global user.name "TuNombre"
git config --global user.email "tu_email.com"
  4. Verifica tu configuración: 
    git config --list

📦 Crear y conectar repositorios

Desde GitHub puedes crear un repositorio en segundos. Puedes inicializarlo con un README o un archivo .gitignore. Luego, puedes conectar tu proyecto con este repo remoto: 

git init
git remote add origin https://github.com/usuario/repositorio.git
git add .
git commit -m "initial commit"
git push -u origin main

💡 ¿Qué es el .gitignore?

Es un archivo donde indicas qué archivos o carpetas no quieres subir a GitHub, como configuraciones locales o archivos sensibles.

💻 Comandos básicos

  • git init: Inicializa un repositorio local.
  • git clone URL: Clona un repo remoto.
  • git status: Verifica el estado de tu proyecto.
  • git add .: Añade todos los archivos al área de preparación.
  • git commit -m "mensaje": Crea un snapshot de los cambios.
  • git push: Sube tus commits al repo remoto.
  • git pull: Descarga cambios del repo remoto y los une con tu rama actual.
  • git branch: Muestra las ramas disponibles.
  • git checkout nombre-rama: Cambia de rama.
  • git log --oneline --graph: Muestra el historial en forma de árbol.

🌱 Estados de los archivos en Git

  • Untracked: archivos nuevos no añadidos.
  • Modified: archivos editados pero no añadidos al staging.
  • Staged: archivos listos para ser confirmados.
  • Committed: cambios guardados en el historial.

📘 Git Stash

Guarda cambios temporales sin hacer commit:

git stash
                                        # Recuperar:
                                        git stash apply
                                        # Eliminar:
                                        git stash drop

🧠 GitFlow - Flujo de trabajo recomendado

GitFlow organiza el trabajo en ramas con diferentes propósitos:

  • main: código estable en producción.
  • development: integración de nuevas funcionalidades.
  • feature/: nuevas funcionalidades que parten de development.
  • release/: preparación para producción.
  • bugfix/: corrección de errores antes de producción.
  • hotfix/: solución urgente en producción.

🧨 Resolución de conflictos

Cuando dos ramas modifican el mismo archivo, Git lanza un conflicto. Debes editar el archivo, resolver el conflicto y luego: 

git add archivo
git commit -m "conflicto resuelto"

🔍 Alternativas modernas

  • git switch → en vez de checkout para cambiar ramas.
  • git restore → para descartar cambios en archivos.

🎥 Interfaz gráfica (VS Code + GitLens)

Con extensiones como GitLens puedes ver el historial de cada archivo, comparar ramas y hacer commits sin usar terminal.

🎯 Buenas prácticas

  • Haz commits pequeños y frecuentes.
  • Usa mensajes claros: feature: navbar, fix: bug botón.
  • No trabajes directamente en main.
